・ 在室中のメンバー一覧を表示可能にする。
　※個人情報保護、機密情報保持の観点から、入室者一覧をいつでも誰でも一覧で表示できるようにするのは問題。
　※現時点においても強制退出の仕様があるため、それほどニーズは高くないともいえる。
　※管理ユーザは在室者一覧を表示出来た方がよいと思う。又、一覧から在室者を強制退出させるなどもできた方がよさそう。
・ 管理ユーザ画面に遷移するためのパスワード制限を設ける。
　※管理者用カードを通した場合に管理者モードに移行するようにすればよいと考える。
・ メンバー（社員）が多い場合やシステムの堅牢性に配慮すると、データベースを導入するべき。
　※現状は、メンバー数が少ないことから、現設計としている。
・登録者リストに登録する社員番号や社員名の文字数、文字種の制限について、現状はかけていない。
　※どういうのをエラーにするか、考えてから、必要に応じて対応する。
・保存したログをメールで送信する機能が欲しい。そのとき、送信するログはユーザが選択できるようにする。
　※使用するユーザはネットワーク経由でラズパイにアクセスするスキルが無いかもしれない。
・登録者リストの重複チェック機能が欲しい。
・IDや社員氏名での検索もしくはフィルタ機能が欲しい。
・IDや社員氏名でのソート機能が欲しい。
・管理ユーザ画面においてボタン押下を無効な状況においては押下ができないようにする。
